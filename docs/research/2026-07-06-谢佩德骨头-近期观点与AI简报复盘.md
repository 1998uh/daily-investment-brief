## 值得学习的观点

| 观点 | 对本项目的启发 |
|---|---|
| Prompt 写得越细，日报质量可能越僵化。 | 每日简报主模板应该规定底线和必填字段，但正文栏目要允许动态组织，避免把当天最重要事件塞进固定框。 |
| 稳定数据源比 WebSearch 更重要。 | 宏观、行情、机构观点等高频数据应优先脚本化和接口化；搜索只做补充，不能成为主流程。 |
| 取数和写作要分离。 | 数据采集角色输出结构化 JSON，写作角色只基于已审计的数据生成报告，减少注意力被取数过程耗尽。 |
| 好产出要沉淀，但 memory 不能成为大杂烩。 | 本项目应把长期规则放进少量结构化文档和模板，避免每次全量加载历史偏好、观点和旧任务。 |
| 角色扮演不是关键，重开上下文和清晰任务边界才关键。 | 不要迷信“CIO/CTO/CRO”人格设定；更重要的是输入、输出、停止条件、审计规则和责任边界。 |
| 功能要脚本化，而不是提示词化。 | 数据抓取、清洗、去重、覆盖统计、上期关注提取都应该由代码完成，LLM 负责解释、归纳和写作。 |
| AI 更像顾问，不是员工。 | 使用 AI 时应让它参与分析、质疑、补充框架，而不是指望它在无工具状态下独立完成复杂流程。 |
| 黑箱模型的结论要单独标注局限。 | DSGE、因子模型、情绪模型等输出必须说明方法、滞后性、无法捕捉的变量，以及和主观框架的分歧。 |
| 投资判断要抓“核心未决问题”。 | 江波龙报告里真正关键不是 2026H1 高增长，而是 2027 年是超级周期延续还是周期回落。简报应突出这种变量。 |
| AI 对投资业的冲击是效率提升，不必自动等同于价值创造。 | 写 AI 产业观点时，要区分“降本增效”“新增需求”“利润归属”“股价是否已计价”。 |

## 他 AI 简报里可以复制的结构

### 1. 宏观周报

可复制点：

- 用“数据覆盖与审计”放在前面，先告诉读者哪些数据完整、哪些缺失、哪些过期。
- 每个宏观变量都输出“当前值、趋势、信号、解释、来源、滞后性”。
- 中美分开看，再做跨境传导：利率、通胀、信用、流动性、汇率、就业、地产、消费。
- 资产映射不要只给结论，要给驱动因子和置信度。
- 行业矩阵可以用“宏观因子 x 行业”的表格，而不是空泛写顺周期/成长/防御。
- 主观框架和模型框架要并列：一致处增强信心，分歧处形成下期关注。
- 模型局限必须写出来，例如 Kalman Filter 对单月突变反应慢、DSGE 不捕捉日内流动性指标。

已沉淀到模板：`templates/macro_weekly_reference_prompt.md`

### 2. 个股机构观点汇总

可复制点：

- 报告头部给出：报告日期、股价、市值、覆盖机构数量、数据交叉验证可靠性。
- 先列公司公告和业绩预告，再列机构观点，避免把研报判断和公告事实混在一起。
- 国内券商和海外投行分表，评级、盈利预测、目标价、核心判断分列。
- 必须有“多空逻辑全景对比”，分别标注来源。
- 找到最关键的未决变量，例如江波龙是“2027 年会怎样”。
- 将行业价格、供需、平台情绪、股吧/雪球/微博/公众号差异放到同一张框架里。
- 风险按严重程度分层，尤其是存货、现金流、负债、内部人减持等和财务报表绑定的风险。

已沉淀到模板：`templates/stock_institution_consensus_prompt.md`

### 3. 对当前每日简报的改造建议

- 保留你已有的“博主持仓信息、看好方向汇总、今日注意事项、下期关注、金句”结构。
- 增加“来源可信度/数据形态”字段：原文观点、机构数据、公司公告、模型输出、编辑归纳要分开标。
- 对高密度个股材料自动提升为“专题章节”，避免埋在普通微博汇总里。
- 在“下期关注”里多写可验证变量，而不是泛泛的板块方向，例如“2027 盈利预测是否继续下修”“存储现货涨价是否放缓”“内部人减持是否延续”。
- 对模型或 AI 生成内容加“黑箱/未核验/需交叉验证”标签，尤其当博主本人也说无法验证时。

## 可执行落地清单

1. 把宏观周报从“日报模板”中独立出来，使用 `templates/macro_weekly_reference_prompt.md` 做专项周报。
2. 对任何单个股票的高密度机构材料，使用 `templates/stock_institution_consensus_prompt.md` 生成专题，而不是混进普通日报。
3. 后续 pipeline 可增加一个轻量规则：如果同一篇源文包含“机构、评级、目标价、净利润、风险、分歧”等字段，则标记为 `standalone_worthy=true`。
4. 每次生成报告时保存中间 JSON，并把缺失字段、异常值、来源冲突写入报告，不要让 LLM 静默补齐。
5. 控制 memory 入口：只加载当前任务相关模板和最近一期观察点，不全量加载历史项目记忆。
